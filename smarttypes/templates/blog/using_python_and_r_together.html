Using python and R together

i'm a python dev, learning R -- python has it's strengths, R has it's strengths (more on this below)

after reading PypeR, A Python Package for Using R in Python
i decided file-based integration was the best way to go

python writes everything to an easily digestible file -- r reads it, does some crunching and writes to it's own file

the problem w/ tools like rpy is memory consumption -- this is from the paper referenced above:

RPy2, a recently developed branch of RPy, uses Python objects to refer to R objects instead of copying them back into Python objects. This strategy avoids frequent data conversions and improves speed. However, memory consumption remains a problem.

OK -- let's address the bigger question, why python + R -- the following is from Ryan Rosario's rpy2 la meetup presentation:
http://www.bytemining.com/wp-content/uploads/2010/10/rpy2.pdf

Rosario asks *and answers* the question
'So, Why use python instead of with R':

  • python has a much larger community with very diverse interests, and text and web mining are big focuses.
  • more access to datastores; not only MySQL and PostgreSQL but also Redis and MongoDB, with more to come.
  • more natural object-oriented features
