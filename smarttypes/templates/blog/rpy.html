RPy2 is a rewrite of RPy, providing for a more user friendly experience and expanded capabilities

Rosario asks *and nicely answers* the question
'So, Why use python instead of with R':

  • python has a much larger community with very diverse interests, and text and web mining are big focuses.
  • state of art algorithms for text mining typically hit python or Perl before they hit R.
  • parallelism does not rely on R's parallelism package
  • more access to datastores; not only MySQL and PostgreSQL but also Redis and MongoDB, with more to come.
  • more natural object-oriented features

I'm a python dev just learning R -- so why the hell would i use R -- Rosario answers that as well
'And, What are the Advantages of R over python':

  • obviously, great analysis and visualization routines.
  • data frames; closest equivalent in python is a dictionary, whose values are the lists of values for each variable, or a list of tuples, each tuple a row.
  • factors; python does not even have an enum primitive data type
